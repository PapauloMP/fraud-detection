# Language Model-Assisted Fraud Detection in Essays

This repository contains the source code, datasets, and engineering pipeline for a machine learning research project focused on detecting AI-assisted text generation in educational environments. 

The project introduces a multiclass classification system designed to distinguish between three authorship categories in Brazilian Portuguese argumentative-expository essays (ENEM format):
1. **Human Authorship:** Authentic student essays.
2. **AI Generation:** Zero-shot synthetic texts generated by high-performance Large Language Models (LLMs).
3. **Copy-Typing (Hybrid Authorship):** AI-generated texts subjected to a controlled algorithmic corruption process that simulates human obfuscation techniques (e.g., typos, structural reordering, and vocabulary simplification).

The system utilizes a fine-tuned BERTimbau model, integrated with Explainable AI tools (SHAP, LIME, and Captum Integrated Gradients) to provide mathematical proofs of the classification decisions.

## Project Structure

### `inputs/`
This directory holds all the foundational data, themes, and prompt engineering instructions required to reproduce the dataset generation and model training.

#### 1. `datasets/`
Contains the final *corpora* used for training, testing, and validation.
* **[Read the detailed dataset documentation here](inputs/datasets/README.md)**.

#### 2. `topics.csv`
A consolidated list containing **151 unique essay themes**. These themes were extracted from the public Essay-BR dataset and cover a wide range of social, political, cultural and others discussions. They were used to prompt the LLMs, ensuring that the synthetic dataset mirrors the thematic distribution of the human control group.

#### 3. `prompts.csv`
Contains the four primary system prompts developed for the data synthesis phase. To mitigate the homogeneity of AI-generated text and introduce representative linguistic variability, the prompts were engineered to simulate distinct student personas and a specialized text-corruption behavior:

* **Prompt 1 (`specialist`):** Instructs the LLM to produce essays with cultured and low-frequency vocabulary, elaborated syntax, diverse socio-cultural repertoire, and sophisticated cohesive mechanisms.
* **Prompt 2 (`good`):** Configures the generation for a secure formal domain and clear articulation of ideas, utilizing conventional phrasal constructions, standard vocabulary without lexical preciousness and pertinent repertoire.
* **Prompt 3 (`average`):** Simulates an intermediate proficiency level, resulting in linear and predictable argumentation, short periods, and basic connectives. The socio-cultural repertoire tends to be generic, superficial, or merely cited without analytical depth.
* **Prompt 4 (`copy_typing`):** A cross-model prompt used exclusively to generate the hybrid class. It restricts the LLM from rewriting the text, forcing it instead to inject human-like noise into an existing synthetic essay (e.g., simplifying connectives, superficial paraphrasing, breaking fluency, and inserting mechanical keyboard typos).

### `src/`
This directory contains the core Python scripts for the project's pipeline, covering data engineering, model training, and explainability.

* **`generation/`**: A subdirectory dedicated to the automated generation of the synthetic (AI) and hybrid (Copy-Typing) classes for the dataset.
* **Core Scripts:** Includes the main scripts for fine-tuning the classification model, executing inferences, extracting linguistic features, and conducting Explainable AI analyses.
